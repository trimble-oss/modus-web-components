# Textarea Input

---

[Modus Text](https://modus.trimble.com/components/inputs/) Input web components are wrappers around native `<textarea>` elements. They are referenced using the `<modus-textarea-input>` custom HTML element.

This component is compatible with Angular reactive forms. This can be achieved through the implementation of a simple directive.

### Properties

| `clearable`                 | `clearable`                    | (optional) Whether the textarea has a clear button.                                                       | `boolean`                                                                   | `false`     |
| `disabled`                  | `disabled`                     | (optional) Whether the textarea is disabled.                                                              | `boolean`                                                                   | `undefined` |
| `enterkeyhint`              | `enterkeyhint`                 | Which action label to present for the enter key on virtual keyboards.                                     | `"enter", "done", "go", "next", "previous", "search", "send"`               | `undefined` |
| `errorText`                 | `error-text`                   | (optional) The textarea's error state text.                                                               | `string`                                                                    | `undefined` |
| `helperText`                | `helper-text`                  | (optional) The textarea's helper text displayed below the input.                                          | `string`                                                                    | `undefined` |
| `label`                     | `label`                        | (optional) The textarea's label.                                                                          | `string`                                                                    | `undefined` |
| `maxLength`                 | `max-length`                   | (optional) The textarea's maximum length.                                                                 | `number`                                                                    | `undefined` |

| `placeholder`               | `placeholder`                  | (optional) The textarea's placeholder text.                                                               | `string`                                                                    | `undefined` |

### DOM Events

| Name          | Description                          | Emits    |
| ------------- | ------------------------------------ | -------- |
| `valueChange` | Fires on textarea input value change | `string` |

### Methods

| Method name  | Description     | Parameter | Return          |
| ------------ | --------------- | --------- | --------------- |
| `focusInput` | Focus the input |           | `Promise<void>` |

### Accessibility

- Textarea Input gets an `aria-label` provided by the `aria-label` property input.
- It is recommended to pass `aria-label` when there is no label used so the input is accessible to screen readers.
- Textarea Input gets an `aria-disabled` set to whether Textarea Input is disabled.
- Textarea Input gets an `aria-placeholder` provided by the `placeholder` property input.
- Textarea Input gets an `aria-invalid` set to whether Textarea Input is invalid.
- Textarea Input gets an `aria-readonly` set to whether Textarea Input is readonly.
- Textarea Input gets an `aria-required` set to whether Textarea Input is required.
