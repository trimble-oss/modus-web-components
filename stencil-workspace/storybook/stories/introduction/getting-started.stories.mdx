# Getting Started

The Modus Web Components package is hosted publicly through the npm package registry. To consume this package, you'll need to have the [npm CLI](https://docs.npmjs.com/downloading-and-installing-node-js-and-npm) installed.

## Installation

1. From the root of your application, run the command:

```
npm install @trimble-oss/modus-web-components --save
```

2. Make sure Open Sans, and Google Material Icons are available browser resources in your application:

```html
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css?family=Open+Sans:300,400,600,700&display=fallback"
  rel="stylesheet" />
<link
  href="https://fonts.googleapis.com/icon?family=Material+Icons&display=block"
  rel="stylesheet" />
```

3. The package exposes the `defineCustomElements()` method which will define every component in the bundle. It does not run automatically, so you must import it and execute it yourself. We recommend you do this near the root of your application like so:

```javascript
import { defineCustomElements } from '@trimble-oss/modus-web-components/loader';

defineCustomElements();
```

- You can now reference the Modus Web Components like any other standard HTML element:

```html
<modus-alert message="You've installed Modus Web Components!" type="success" />
```

## Recommendations

- Abstract away from 3rd party libraries. Consider wrapping each Modus Web Component in a component native to your application.
